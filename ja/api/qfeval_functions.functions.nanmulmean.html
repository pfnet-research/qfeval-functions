qfeval_functions.functions.nanmulmean

nanmulmean(x, y, dim=(), keepdim=False, *, _ddof=0)[ソース]

メモリ効率に優れた方法で、NaN値を無視した要素ごとの積の平均値を計算します。

この関数は、中間的な積テンソルをメモリ上に作成することなく、2つのテンソルの要素ごとの積の平均値 nanmean(x * y, dim) を計算します。また、計算から NaN 値を除外します。もし x または y の特定の位置に NaN 値がある場合、そのペアは平均値の計算から除外されます。

この関数は数学的には nanmean(x * y, dim) と等価ですが、完全な積テンソルをメモリ上に展開しないよりメモリ効率の高い実装を採用しており、大規模なテンソル演算や複雑なブロードキャストパターンに適しています。

NaNを考慮した平均値は以下のように計算されます：

\[\text{nanmulmean}(X, Y) = \frac{\sum_{i \text{ valid}} X_i \cdot Y_i}{ N_{\text{valid}} - \text{ddof}}\]

ここで、和は有効な（NaNでない）ペアに対して計算され、:math:`N_{ ext{valid}}`は有効なペアの数を表します。

パラメータ:

  • x (Tensor) -- 最初の入力テンソル。

  • y (Tensor) -- 第2の入力テンソル。 x とブロードキャスト可能である必要があります。

  • dim (int or tuple of ints, optional) -- 平均を計算する次元を指定します。指定しない場合（デフォルトは空のタプル）、全次元にわたって平均が計算されます。

  • keepdim (bool, optional) -- 出力テンソルが dim を保持するかどうか。デフォルトは False。

  • _ddof (int, optional) -- 内部計算で使用する自由度のデルタ値。除数は``N_valid - _ddof``で表され、``N_valid``は有効な（NaNでない）ペアの数です。デフォルト値は0です。これは内部パラメータです。

戻り値:

有効な（NaNでない）ペアのみを対象に計算された要素ごとの積の平均値。形状は入力次元、dim、および keepdim パラメータに依存します。

戻り値の型:

Tensor

サンプル

>>> # Simple element-wise product mean with NaN
>>> x = torch.tensor([1.0, 2.0, nan, 4.0])
>>> y = torch.tensor([2.0, nan, 4.0, 5.0])
>>> QF.nanmulmean(x, y)
tensor(11.)

>>> # 2D tensors with NaN values
>>> x = torch.tensor([[1.0, nan, 3.0],
...                   [4.0, 5.0, nan]])
>>> y = torch.tensor([[2.0, 4.0, nan],
...                   [nan, 10.0, 12.0]])
>>> QF.nanmulmean(x, y, dim=1)
tensor([ 2., 50.])

>>> # Broadcasting with NaN handling
>>> x = torch.tensor([[1.0], [nan], [3.0]])
>>> y = torch.tensor([2.0, 4.0, nan])
>>> QF.nanmulmean(x, y)
tensor(6.)

>>> # With keepdim
>>> x = torch.tensor([[1.0, nan], [3.0, 4.0]])
>>> y = torch.tensor([[2.0, 4.0], [nan, 5.0]])
>>> QF.nanmulmean(x, y, dim=1, keepdim=True)
tensor([[ 2.],
        [20.]])
